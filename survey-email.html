Survey APIs (Public)

Tài liệu API tích hợp khảo sát dành cho MeviFarm & Bên thứ 3. (Không yêu cầu Authentication)

Yêu cầu Header bắt buộc (Global Headers)

Tất cả các request gọi tới API Survey bắt buộc phải truyền header origin như bên dưới. FE cần cấu hình ngay trong Axios Interceptors (hoặc tương đương) để mọi request đều đính kèm header này.

origin: https://mevi.edtexco.vn

Môi trường & Base URL

Tất cả các API endpoint bên dưới đều được nối tiếp vào Base URL này. Vui lòng sử dụng đúng domain theo môi trường tích hợp của bạn.

https://api.xbest.io/lms-survey-service/v1 Production

Xem chi tiết bài khảo sát

Lấy thông tin định danh người dùng và danh sách câu hỏi trong kỳ khảo sát thông qua email. Cung cấp dữ liệu để Frontend dựng giao diện Form khảo sát.

GET /survey-periods/{surveyPeriodId}/results

Request Parameters

Parameter In Type Description
surveyPeriodIdRequired path integer ID của kỳ khảo sát. Ví dụ: 395
emailRequired query string Email của người tham gia khảo sát (định danh).

Response Schema (Chi tiết các trường)

Phân tích cấu trúc JSON trả về. Các trường đánh dấu Enum rất quan trọng để quyết định logic hiển thị UI trên Frontend.

Field Type Description & Values
code number

Mã trạng thái trả về từ Application. (Ví dụ: 200 là thành công)

message string

Thông báo dạng text từ server (hiển thị dạng Toast/Alert nếu cần).

data object

Payload chính chứa toàn bộ thông tin chi tiết của bài làm (Survey Result).

id integer

ID định danh bài làm khảo sát của User này.
companyId integer

Mã công ty (Định danh tenant/khách hàng trong hệ thống multi-tenant).

userId string(UUID)

ID định danh tài khoản người dùng trên hệ thống LMS.

userName string

Tên hiển thị của người dùng (dùng để in ra lời chào: "Chào anh/chị X").

email string

Email của người làm bài (khớp với email truyền lên).

status string (Enum)

[Quan trọng] Trạng thái bài làm của người dùng đối với kỳ khảo sát này:

  • not_started: Chưa nộp bài (Frontend hiển thị form cho phép làm bài).
  • submitted: Đã nộp bài (Frontend hiển thị màn hình báo "Bạn đã hoàn thành khảo sát", khóa form).
surveyPeriodName string

Tên hiển thị của kỳ khảo sát (Hiển thị làm Title màn hình).

submittedAt string (Date) | null

Thời gian nộp bài. Bằng null nếu status là not_started.

resultQuestions array[object]

Danh sách toàn bộ các câu hỏi nằm trong kỳ khảo sát này.

id integer

ID của câu hỏi. Cần lưu lại để map vào questionId khi gọi API POST nộp bài.

code string

Mã code của câu hỏi (Thường dùng để tracking, đánh index hoặc hiển thị mã câu hỏi nếu cần).

companyId integer

Mã công ty phân bổ cho câu hỏi này.
type string (Enum)

[Định tuyến UI] Loại câu hỏi. Dựa vào đây FE quyết định render Input, Radio, Checkbox hay Matrix:

  • select: Câu hỏi lựa chọn 1 đáp án.
  • multi_select: Câu hỏi lựa chọn nhiều đáp án.
  • essay: Tự luận.
  • linear_range: Câu hỏi phạm vi tuyến tính (ví dụ: đánh giá 1-5 sao).
  • single_choice_matrix: Câu hỏi ma trận một lựa chọn (Radio Matrix).
  • multi_choice_matrix: Câu hỏi ma trận nhiều lựa chọn (Checkbox Matrix).
  • linear_matrix: Câu hỏi ma trận tuyến tính.
content string

Nội dung câu hỏi dạng text thuần (dùng nếu không hỗ trợ HTML).

contentRich string

Nội dung câu hỏi có format HTML (VD: có in đậm, xuống dòng, thẻ <p>). FE ưu tiên dùng trường này để render bằng v-html hoặc dangerouslySetInnerHTML.

answers array[object]

Danh sách các cấu trúc đáp án. Chứa id bắt buộc phải gửi lại server khi user trả lời.

id integer

[Rất quan trọng] ID định danh của đáp án này.
Dù là câu Tự luận (essay), hệ thống vẫn sinh sẵn 1 object answer, FE phải lấy `id` này truyền vào mảng `answerIds` khi nộp bài.

content string

Nội dung text của đáp án (Dành cho câu trắc nghiệm hiển thị label "A, B, C, D"). Nếu là câu `essay`, trường này thường rỗng từ server trả về.

isOther boolean | null

Cờ xác định đáp án có phải dạng "Khác (vui lòng điền rõ)" hay không. Nếu `true`, FE cần render thêm 1 ô input text đi kèm khi user chọn đáp án này.

Nộp bài khảo sát

API ghi nhận kết quả làm bài của User. Yêu cầu truyền email trong body để đảm bảo tính định danh và toàn vẹn dữ liệu. Cấu trúc dataSubmit được map dựa trên ID câu hỏi và ID đáp án đã lấy từ API GET.

POST /survey-periods/{surveyPeriodId}/results/submit

Path Parameters

Parameter Type Description
surveyPeriodIdRequired integer ID của kỳ khảo sát

Body Parameters (application/json)

Parameter Type Description & Rules
emailRequired string

Email định danh của người nộp bài. Phải khớp với email đã gọi ở API GET.

dataSubmitRequired array[object]

Mảng chứa danh sách kết quả làm bài của từng câu hỏi.

questionIdRequired integer

ID của câu hỏi. Lấy từ resultQuestions[i].id ở API GET.

answersRequired array[object]

Mảng chứa chi tiết dữ liệu người dùng trả lời cho câu hỏi này.

answerIdsRequired array[integer]

Mảng chứa ID của các đáp án người dùng đã chọn. (Lấy từ resultQuestions[i].answers[j].id).

  • Trắc nghiệm 1 đáp án: Truyền mảng 1 phần tử VD: [1824]
  • Trắc nghiệm nhiều đáp án: Truyền mảng nhiều phần tử VD: [1824, 1825]
  • Tự luận (essay): Vẫn phải truyền mảng chứa ID của answer placeholder được server trả về ban đầu.
content string

Nội dung text do người dùng điền vào.

  • Bắt buộc nếu câu hỏi là dạng Tự luận (essay).
  • Bắt buộc nếu user tick vào đáp án có cờ isOther = true (Mục "Khác...").
  • Các trường hợp chọn trắc nghiệm thông thường có thể để chuỗi rỗng "" hoặc lược bỏ.

Bảng mã lỗi (Error Codes)

Phân biệt rõ hai nhóm lỗi chính mà API có thể trả về khi nộp bài không thành công.

Status Code Loại lỗi Nguyên nhân Hành động FE
400 Bad Request

FE truyền sai cấu trúc dữ liệu:

  • Thiếu field bắt buộc (email, dataSubmit...)
  • Sai kiểu dữ liệu (VD: answerIds truyền string thay vì array)
  • Body rỗng hoặc không phải JSON hợp lệ

Kiểm tra lại payload trước khi gửi. Đây là lỗi do FE code sai, cần fix ở phía client.

422 Unprocessable Entity

Dữ liệu đúng cấu trúc nhưng vi phạm logic nghiệp vụ phía server:

  • User đã nộp bài rồi (duplicate submission)
  • Kỳ khảo sát đã hết hạn hoặc bị đóng
  • Email không thuộc danh sách được phép tham gia

Hiển thị thông báo message từ response body dưới dạng Toast/Alert cho người dùng. Không cần retry.
